# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build & run

```sh
bash build.sh          # compile → ./build/Key54.app (ad-hoc signed)
bash install.sh        # build + copy to /Applications + launch
bash screenshot.sh     # capture a fresh settings-window PNG to screenshots/
```

`install.sh` also refreshes `screenshots/settings.png` after install. The `--screenshot <path>` CLI flag drives this: it renders the settings window over a neutral backdrop, captures it, and quits (requires Screen Recording permission).

To release: push a `v*` tag. GitHub Actions (`.github/workflows/release.yml`) builds on `macos-26`, packages a DMG, and optionally signs + notarizes if the repo secrets are set.

## Architecture

Everything lives in one file: **`main.swift`** (~1580 lines). No Xcode project — built directly with `swiftc`. The entry point is at the bottom of the file (imperative, not `@main`).

### Key types

- **`AppDelegate`** — the application's nerve centre. Installs a `CGEventTap` on `flagsChanged` watching keycode 54 (right ⌘). Implements a two-stage trigger: a *buffer* timer (Key Delay dead-zone) followed by an *arm* phase (charge animation + hold timer). Manages app-switch logic via `NSWorkspace` and the Accessibility API for full-screen edge cases. Persists all settings to `UserDefaults`.

- **`SettingsWindow`** — built entirely in code, no NIB. The whole layout is regenerated by `rebuild()` whenever the preset changes (because the Custom sub-panel and Animation Style selector appear/disappear). Uses numeric `tag` values to find the icon/name views without storing explicit `IBOutlet`-style references. The static tables `keyDelayValues` / `animationLengthValues` / `holdDurationLabels` are the source of truth for all presets.

- **`TriggerHUD`** — a borderless, click-through `NSPanel` at `.screenSaver` level that floats above all Spaces. Animates with CALayer: a `CAShapeLayer` ring sweep (Power Up) or a rising fill body clipped to a rounded rect (Level Up). Uses a `fadeGeneration` counter to safely cancel in-flight completions when a new hold starts.

- **`StyleGlyph` / `StylePill`** — pure-CALayer mini-previews of each animation style, rendered as static ¾-filled glyphs in the settings window.

- **`LinkButton` / `HoverButton`** — `NSButton` subclasses for the pointing-hand cursor and hover tint, used in the Tip Jar popover.

### UserDefaults keys

| Key | Type | Default |
|---|---|---|
| `targetAppPath` | String | Terminal.app |
| `presetIndex` | Int | 2 (Medium) |
| `customKeyDelay` | Double | 0.5 |
| `customAnimationLength` | Double | 0.4 |
| `animationStyle` | Int | 0 (Power Up) |
| `showAnimation` | Bool | true |

### Liquid Glass (macOS 26+)

The HUD backdrop uses `NSGlassEffectView` on macOS 26+, falling back to `NSVisualEffectView(.hudWindow)`. This is gated with `#if compiler(>=6.2)` (SDK availability) and `@available(macOS 26.0, *)` (runtime). CI runs on `macos-26` so release builds include the glass path; local builds on older SDKs get the frosted fallback automatically.

### Icon generation

`make_icon.swift` is a standalone Swift script (run by `build.sh`) that draws all icon sizes programmatically. `headshot.heic` is copied into `Resources` for the Tip Jar popover.

## Required permissions

- **Accessibility** — needed for the CGEventTap. Without it, the tap can't be created; the app falls back to watching `com.apple.accessibility.api` notifications and retries when the user grants access.
- **Screen Recording** — needed only for `--screenshot` capture mode.
